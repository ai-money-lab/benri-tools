# ROCKEDGE AI-OS 週次SOP (Standard Operating Procedure)

## HIROKIの週次作業: 月曜日に5分だけ

### 月曜朝のルーティン

1. **メール通知を確認**（1分）
   - GitHub Actions の実行結果メール
   - 新規データ件数の確認

2. **`data/kpi_history.json` を確認**（2分）
   - 総データ数の推移
   - 週間増加数
   - 平均家賃比率の変化

3. **投稿カレンダーを確認**（1分）
   - `data/posting_calendar.md` で今週の投稿を確認
   - TikTok台本を `data/latest_content.md` から取得

4. **DECISION: 1つだけ判断**（1分）
   - AI-OSから提示される YES/NO 質問に回答
   - 回答方法: GitHubのIssueにコメント or Slack

## KPIトラッキング

### Phase判定基準

| Phase | データ数 | 月間収益目標 | HIROKI作業 |
|-------|---------|-------------|-----------|
| 1 (現在) | 0-100 | ¥0 | 週5分 |
| 2 | 100-500 | ¥30,000-150,000 | 週5分 |
| 3 | 500-2000 | ¥150,000-500,000 | 月30分 |
| 4 | 2000+ | ¥500,000+ | 月15分 |

### 週次KPI確認項目

1. **データ数**: `data/raw_data.json` → totalEntries
2. **新規データ**: `data/raw_data.json` → newThisWeek
3. **投稿数**: 手動カウント
4. **エンゲージメント**: X Analytics
5. **note販売数**: note ダッシュボード
6. **推定月間収益**: note売上 + アフィリエイト

## 異常検知ルール

### 自動アラート条件
- データ増加が0件/週 → 集客施策を見直し
- 投稿エンゲージメントが前週比50%以下 → コンテンツ見直し
- サイトがダウン → デプロイ確認

### エスカレーション条件（HIROKIに即報告）
- セキュリティインシデント
- 法的リスクのあるコンテンツ検出
- コストが¥10,000を超える提案
